# ROS2 第一课：最小发布者/订阅者

## 前提
- WSL2 + Ubuntu 22.04 + ROS2 Humble 已安装
- 每次打开新终端，先 source ROS2 环境：
  ```bash
  source /opt/ros/humble/setup.bash
  ```
  （建议加到 `~/.bashrc` 里一劳永逸）

## 步骤 1：把代码复制到 WSL

在 WSL 终端里执行（把路径替换成你的 Windows 用户名）：

```bash
cp -r /mnt/c/Users/Akko/WorkBuddy/2026-06-22-23-55-56/ros2_ws/src/my_first_pkg ~/ros2_ws/src/
```

如果你还没有 ~/ros2_ws 目录，先创建：
```bash
mkdir -p ~/ros2_ws/src
cp -r /mnt/c/Users/Akko/WorkBuddy/2026-06-22-23-55-56/ros2_ws/src/my_first_pkg ~/ros2_ws/src/
```

## 步骤 2：编译

```bash
cd ~/ros2_ws
colcon build --packages-select my_first_pkg
```

编译成功后，source 工作空间：
```bash
source ~/ros2_ws/install/setup.bash
```

## 步骤 3：运行 publisher（终端 1）

```bash
ros2 run my_first_pkg publisher
```

预期输出：
```
[INFO] [minimal_publisher]: Publishing: 'Hello, world! 0'
[INFO] [minimal_publisher]: Publishing: 'Hello, world! 1'
[INFO] [minimal_publisher]: Publishing: 'Hello, world! 2'
...
```

## 步骤 4：运行 subscriber（终端 2）

新开一个 WSL 终端，记得 source：
```bash
source /opt/ros/humble/setup.bash
source ~/ros2_ws/install/setup.bash
ros2 run my_first_pkg subscriber
```

预期输出：
```
[INFO] [minimal_subscriber]: I heard: 'Hello, world! 0'
[INFO] [minimal_subscriber]: I heard: 'Hello, world! 1'
[INFO] [minimal_subscriber]: I heard: 'Hello, world! 2'
...
```

## 步骤 5：验证通信

在第三个终端用 ROS2 CLI 工具观察：

```bash
# 查看所有活跃节点
ros2 node list
# 预期看到：
#   /minimal_publisher
#   /minimal_subscriber

# 查看所有话题
ros2 topic list
# 预期看到：
#   /topic

# 实时查看话题数据
ros2 topic echo /topic
# 预期看到每 500ms 一条消息

# 查看话题信息
ros2 topic info /topic
# 预期看到：1 publisher, 1 subscriber
```

## 成功标准

- [ ] publisher 终端持续输出 "Publishing: 'Hello, world! N'"
- [ ] subscriber 终端持续输出 "I heard: 'Hello, world! N'"
- [ ] `ros2 topic echo /topic` 能看到消息流
- [ ] Ctrl+C 能干净退出

## 代码结构解读

```
my_first_pkg/
├── CMakeLists.txt          # 告诉 colcon 怎么编译
├── package.xml             # 声明包名、依赖、构建工具
└── src/
    ├── minimal_publisher.cpp   # 发布者节点
    └── minimal_subscriber.cpp  # 订阅者节点
```

### 核心模式（C++ 节点通用骨架）

```cpp
class MyNode : public rclcpp::Node {
public:
    MyNode() : Node("node_name") {
        // 在构造函数里创建 publisher / subscriber / timer
    }
private:
    // 回调函数
};

int main(int argc, char** argv) {
    rclcpp::init(argc, argv);           // 初始化 ROS2
    rclcpp::spin(std::make_shared<MyNode>());  // 阻塞运行，处理回调
    rclcpp::shutdown();                 // 清理
    return 0;
}
```

几乎所有 ROS2 C++ 节点都遵循这个模式，记住这个骨架就够了。
